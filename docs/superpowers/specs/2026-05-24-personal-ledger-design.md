# 个人云同步记账软件设计

日期：2026-05-24

## 背景

用户需要一个个人使用的记账软件，可以在手机、平板、电脑上轻便使用，并能同步数据。核心需求包括快速记账、查看任意时间段收入支出、显示当前手里还有多少钱，以及尽量关联微信和支付宝数据。

该产品首版定位为个人账本，不做家庭共享、店铺经营账本或多角色权限。微信和支付宝部分采用账单文件导入，不做个人账号后台自动读取。原因是微信支付官方账单 API 主要面向商户号，支付宝对账单下载接口也面向开放平台商户/服务商账单；个人用户更稳定可行的方式是从微信或支付宝导出账单文件后导入。

参考资料：

- 微信支付商户文档，申请交易账单：https://pay.wechatpay.cn/doc/v3/merchant/4012791907
- 支付宝开放平台，alipay.data.dataservice.bill.downloadurl.query 查询对账单下载地址：https://developer.alibaba.com/docs/api.htm?apiId=1054&docType=4

## 目标

首版要完成一个可用的 PWA 网页应用：

- 手机、平板、电脑均可通过浏览器访问，也可添加到主屏幕。
- 单人账号登录，同一账号在多设备之间云同步。
- 首页采用概览优先布局，打开后先看到当前余额、收入、支出和趋势。
- 当前余额按 `初始本金 + 全部收入 - 全部支出` 计算。
- 支持快速记录收入和支出。
- 支持选择任意日期范围，查看总收入、总支出、净额和分类汇总。
- 支持内置常用分类，并允许用户自定义分类。
- 支持导入微信、支付宝导出的账单文件，导入前可预览、去重、修正分类。

## 非目标

首版不做以下内容：

- 多人共享账本、家庭成员权限或协作审批。
- 银行卡、微信、支付宝个人账号的自动登录抓取。
- 商户号自动拉取账单，除非后续用户提供商户号和开放平台接入信息。
- 预算、借贷、投资、发票、报销等复杂财务模块。
- 原生 iOS、Android 或桌面客户端。

## 用户体验

首页优先显示概览：

- 顶部显示当前余额，作为最醒目的主数字。
- 当前余额下方显示总收入、总支出，以及本月或当前筛选周期的变化。
- 首页提供明显的“记支出”和“记收入”入口。
- 最近记录列表展示最新几笔交易，便于快速检查。
- 时间筛选入口允许选择本月、上月、今年、自定义日期范围。

快速记账表单应尽量短：

- 类型：收入或支出。
- 金额：必填。
- 分类：默认展示常用分类，可搜索或新建。
- 日期：默认今天。
- 备注：可选。
- 支付来源：可选，首版可用“现金/微信/支付宝/银行卡/其他”等标签表示。

统计页提供：

- 日期范围选择。
- 总收入、总支出、净额。
- 分类汇总列表。
- 收入/支出切换。
- 明细列表，可按日期倒序查看。

设置页提供：

- 初始本金设置。
- 分类管理。
- 导入账单。
- 数据同步状态。
- 退出登录。

## 数据模型

核心实体如下：

### UserProfile

- `id`
- `email`
- `created_at`
- `updated_at`

### LedgerSettings

- `user_id`
- `initial_principal`
- `currency`
- `created_at`
- `updated_at`

`initial_principal` 是用户手动输入的现有本金。当前余额通过交易记录实时计算，不直接作为可随意编辑的字段保存。

### Category

- `id`
- `user_id`
- `name`
- `type`：`income` 或 `expense`
- `color`
- `icon`
- `is_system`
- `sort_order`
- `created_at`
- `updated_at`

系统预置常用分类，例如：

- 支出：餐饮、交通、购物、住房、生活缴费、娱乐、医疗、学习、旅行、其他。
- 收入：工资、奖金、兼职、退款、转账、投资收益、其他。

### Transaction

- `id`
- `user_id`
- `type`：`income` 或 `expense`
- `amount`
- `category_id`
- `occurred_at`
- `note`
- `source`：`manual`、`wechat_import`、`alipay_import`
- `payment_channel`：`wechat`、`alipay`、`cash`、`bank_card`、`other`
- `external_key`
- `created_at`
- `updated_at`

`external_key` 用于账单导入去重。手动记录可以为空。

### ImportBatch

- `id`
- `user_id`
- `source`：`wechat` 或 `alipay`
- `file_name`
- `status`：`previewing`、`imported`、`failed`
- `total_rows`
- `imported_rows`
- `duplicate_rows`
- `created_at`

## 计算规则

当前余额：

```text
current_balance = initial_principal + sum(all income transactions) - sum(all expense transactions)
```

任意时间段统计：

```text
period_income = sum(income transactions where occurred_at in range)
period_expense = sum(expense transactions where occurred_at in range)
period_net = period_income - period_expense
```

时间范围使用用户本地时区展示和筛选。金额使用十进制定点数处理，避免浮点误差。

## 同步方案

首版采用云同步：

- 用户通过邮箱登录，优先使用邮箱验证码或 magic link，避免用户记额外密码。
- 账本数据存储在云数据库。
- 每条数据都带 `user_id`，只能访问当前用户的数据。
- 应用启动后拉取云端数据。
- 新增、编辑、删除交易后立即写入云端。
- 离线时可以显示已有缓存，写入操作首版可以提示需要联网；后续再扩展离线队列。

推荐技术实现为 Supabase 或同类 Backend-as-a-Service：

- Auth 负责登录。
- Postgres 负责结构化账本数据。
- Row Level Security 确保用户只能访问自己的数据。
- 前端部署为静态 PWA。

如果后续用户希望完全自托管，可以将数据层替换为自建 Postgres + API 服务。

生产环境云同步需要配置 Supabase 项目的 URL 和匿名公钥。开发环境可以先使用本地演示模式完成界面和计算验证，但最终验收同步能力时需要接入真实云项目。

## 微信和支付宝导入

首版导入流程：

1. 用户在微信或支付宝中导出个人账单文件。
2. 在应用中选择来源：微信或支付宝。
3. 上传 CSV 或 Excel 文件。
4. 系统解析文件并展示预览。
5. 系统根据金额正负、交易类型、来源字段推断收入或支出。
6. 系统根据商户名、交易说明、备注等字段自动匹配分类。
7. 用户确认后导入。
8. 导入时按 `source + external_key` 去重。

导入功能需要处理不同版本账单格式变化：

- 解析器按来源拆分为独立模块。
- 未识别字段保留在预览阶段显示。
- 无法判断收入/支出的行需要用户确认。
- 已导入记录默认不重复导入。

## 页面结构

页面分为四个主区域：

- 首页：当前余额、收入支出概览、最近记录、快速入口。
- 记账：新增或编辑交易。
- 统计：任意日期范围汇总、分类汇总、明细筛选。
- 设置：初始本金、分类、导入、账号。

在手机上使用底部导航；在平板和电脑上使用左侧导航加主内容区。

## 错误处理

- 未登录访问账本页面时跳转到登录页。
- 金额为空、为负数或格式错误时阻止提交。
- 分类缺失时提示选择或创建分类。
- 云同步失败时保留表单输入并显示重试入口。
- 导入文件格式不支持时提示用户选择 CSV 或 Excel。
- 导入预览中无法识别的行标记为需要处理，不自动入账。

## 测试重点

首版测试覆盖以下行为：

- 当前余额计算正确。
- 任意日期范围收入、支出、净额计算正确。
- 手动新增、编辑、删除交易后统计更新。
- 内置分类初始化正确，用户可新增自定义分类。
- 导入去重逻辑不会重复写入同一条账单。
- 未登录用户不能读取或写入账本数据。
- 手机、平板、电脑宽度下首页和记账表单可用且不溢出。

## 首版验收标准

- 用户可以创建账号并登录。
- 用户可以设置初始本金。
- 用户可以新增收入和支出记录。
- 首页能显示当前余额，且公式为初始本金加收入减支出。
- 用户可以选择任意日期范围，查看总收入、总支出和净额。
- 用户可以管理分类。
- 用户可以在至少两个设备或浏览器会话中看到同步后的同一份数据。
- 用户可以导入微信或支付宝导出的账单文件，并在确认后生成交易记录。
